## Readme top

**See token risk, wallet behavior, and project context before you swap**

SigmentaAI turns raw on-chain noise into structured AI output across web, Telegram, browser extension, and API

> [!IMPORTANT]
> SigmentaAI is built around real usage flows for Solana traders and builders
> The core journey is simple: analyze → understand risk → decide → swap

## Demo first

Instead of jumping between scanners, feeds, and DEX tabs, SigmentaAI brings the flow into one layer

| Input | AI layer | Outcome |
|---|---|---|
| Token address | Token Analytics + Risk Summary | Clear view of liquidity, volatility, holder concentration, and red flags |
| Wallet address | Wallet Analytics + Behavior Profile | Snapshot of PnL, exposure, trading style, and risk posture |
| Token or project name | Research Agent | Short digest of recent narratives, news, and catalysts |
| Approved trade idea | Jupiter execution flow | One-click path from insight to swap while you keep custody |

```text
Token pasted → metrics loaded → AI summary returned → decision made → Jupiter route opened
```

> [!TIP]
> The product is designed so the user understands the outcome in seconds
> Fast read first, deeper investigation second

## Core use cases

### Token due diligence before entry

Run a token analysis before taking a position
SigmentaAI highlights liquidity quality, volatility, volume behavior, and holder concentration so you can judge whether the setup is tradeable or structurally weak

### Wallet health check and wallet watching

Analyze your own wallet or any public wallet
See how concentrated it is, how it rotates positions, what its rough PnL profile looks like, and whether its behavior appears controlled or reckless

### Research without endless feed scrolling

Use the Research Agent to pull a short digest on a project or token
Instead of manually checking scattered posts and links, you get a compressed brief focused on what changed and why it matters

### Browser-first trading workflow

When you find a token while browsing, the extension can surface context instantly
That shortens the distance between discovery, verification, and action

### API-powered product building

Teams can integrate Analytics Agent and Research Agent through a unified run → job → result flow
This makes SigmentaAI usable as an engine under another product's UI

> [!WARNING]
> Analytics and AI summaries support decision-making
> They do not remove market risk and they are not financial advice

## Product structure

| Layer | Role |
|---|---|
| Token Analytics | Measures market structure, liquidity, volatility, holder concentration, and visible token risk |
| Wallet Analytics | Measures portfolio exposure, PnL patterns, behavior style, and concentration risk |
| Research Agent | Pulls recent project and token context into a short digest |
| Analytics Agent | Returns metrics, flags, scores, and a plain-language summary in a consistent format |
| Jupiter execution flow | Lets users move from validated insight to swap while keeping wallet control |

## Why the model works

SigmentaAI uses a utility token and credits model that ties product usage directly to platform demand

| Component | Function |
|---|---|
| $SIGMENTA | Utility token used to buy credits |
| Credits | Internal usage unit for token scans, wallet scans, and research requests |
| 80% burn | Majority of purchase flow is removed from circulation on-chain |
| 20% treasury | Funds development, infrastructure, ecosystem growth, and future releases |

> [!IMPORTANT]
> SigmentaAI does not custody funds
> Wallet connection is used for access and approved transaction flows
> Signing stays in the wallet
